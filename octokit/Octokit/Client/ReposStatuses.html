      #create_commit_status(repo, sha, state, options = {}) ⇒ Sawyer::Resource

      Create a commit status

      Parameters:

      • repo (Integer, String, Repository, Hash) —

        A GitHub repository

      • sha (String) —

        The sha of the status

      • state (String) —

        The state of the status. Can be one of error, failure, pending, or success.

      • options (Hash) (defaults to: {}) —

        a customizable set of options

      Options Hash (options):

      • :target_url (String) —

        The target URL to associate with this status. This URL will be linked from the GitHub UI to allow users to easily see the source of the status. For example, if your continuous integration system is posting build status, you would want to provide the deep link for the build output for this specific SHA: http://ci.example.com/user/repo/build/sha

      • :description (String) —

        A short description of the status.

      • :context (String) —

        A string label to differentiate this status from the status of other systems.

      Returns:

      • (Sawyer::Resource) —

        The new status

def create_commit_status(repo, sha, state, options = {})
  opts = options.dup
  opts[:state] = state.to_s.downcase
  post "#{Repository.path repo}/statuses/#{sha}", opts
end
