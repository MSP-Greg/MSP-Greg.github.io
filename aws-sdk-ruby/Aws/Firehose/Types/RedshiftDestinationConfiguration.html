      Overview

      Describes the configuration of a destination in Amazon Redshift.

      Returned by:

      • CreateDeliveryStreamInput#redshift_destination_configuration

      Note: When passing RedshiftDestinationConfiguration as input to a Client method, you can use a vanilla Hash:

  },
  username: "Username", # required

      The S3 configuration for the intermediate location from which Amazon Redshift obtains data. Restrictions are described in the topic for CreateDeliveryStream.

      The compression formats SNAPPY or ZIP cannot be specified in RedshiftDestinationConfiguration.S3Configuration because the Amazon Redshift COPY operation that reads from the S3 bucket doesn\'t support these compression formats.
