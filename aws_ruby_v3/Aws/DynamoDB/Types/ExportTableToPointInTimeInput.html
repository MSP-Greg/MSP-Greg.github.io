      #client_token ⇒ String (rw)

      Providing a ClientToken makes the call to ExportTableToPointInTimeInput idempotent, meaning that multiple identical calls have the same effect as one single call.

      A client token is valid for 8 hours after the first request that uses it is completed. After 8 hours, any request with the same client token is treated as a new request. Do not resubmit the same request with the same client token for more than 8 hours, or the result might not be idempotent.

      If you submit a request with the same client token but a change in other parameters within the 8-hour idempotency window, ::Aws::DynamoDB returns an IdempotentParameterMismatch exception.

      A suitable default value is auto-generated. You should normally not need to pass this option.
