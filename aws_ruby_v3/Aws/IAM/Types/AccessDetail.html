      Overview

      An object that contains details about when a principal in the reported AWS Organizations entity last attempted to access an AWS service. A principal can be an ::Aws::IAM user, an ::Aws::IAM role, or the AWS account root user within the reported ::Aws::Organizations entity.

      This data type is a response element in the GetOrganizationsAccessReport operation.

      Instance Attribute Summary

      • #entity_path ⇒ String rw

        The path of the ::Aws::Organizations entity (root, organizational unit, or account) from which an authenticated principal last attempted to access the service.

      • #last_authenticated_time ⇒ Time rw

        The date and time, in [ISO 8601 date-time format][1], when an authenticated principal most recently attempted to access the service.

      • #region ⇒ String rw

        The Region where the last service access attempt occurred.

      • #service_name ⇒ String rw

        The name of the service in which access was attempted.

      • #service_namespace ⇒ String rw

        The namespace of the service in which access was attempted.

      • #total_authenticated_entities ⇒ Integer rw

        The number of accounts with authenticated principals (root users, ::Aws::IAM users, and ::Aws::IAM roles) that attempted to access the service in the reporting period.

      Instance Attribute Details

      #entity_path ⇒ String (rw)

      The path of the ::Aws::Organizations entity (root, organizational unit, or account) from which an authenticated principal last attempted to access the service. AWS does not report unauthenticated requests.

      This field is null if no principals (IAM users, ::Aws::IAM roles, or root users) in the reported ::Aws::Organizations entity attempted to access the service within the reporting period.
