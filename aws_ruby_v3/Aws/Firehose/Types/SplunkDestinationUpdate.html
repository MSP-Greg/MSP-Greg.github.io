    #hec_acknowledgment_timeout_in_seconds  ⇒ Integer  (rw)
  
      

      
  
      
    
      The amount of time that ::Aws::Kinesis Data ::Aws::Firehose waits to receive an
acknowledgment from Splunk after it sends data. At the end of the
timeout period, ::Aws::Kinesis Data ::Aws::Firehose either tries to send the data

    #s3_backup_mode  ⇒ String  (rw)
  
      

      
  
      
    
      Specifies how you want ::Aws::Kinesis Data ::Aws::Firehose to back up documents to
Amazon S3. When set to FailedDocumentsOnly, ::Aws::Kinesis Data ::Aws::Firehose
writes any data that could not be indexed to the configured Amazon
::Aws::S3 destination. When set to AllEvents, ::Aws::Kinesis Data ::Aws::Firehose
delivers all incoming records to Amazon S3, and also writes failed
documents to Amazon S3. The default value is FailedEventsOnly.
      

      You can update this backup mode from FailedEventsOnly to
AllEvents. You can't update it from AllEvents to
FailedEventsOnly.
