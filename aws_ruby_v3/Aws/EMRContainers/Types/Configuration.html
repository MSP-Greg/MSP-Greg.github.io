  properties: {
    "String1024" => "String1024",
  },
  configurations: [
    {

        "String1024" => "String1024",
      },
      configurations: {
        # recursive ConfigurationList
      },
    },
  ],
}

      A configuration specification to be used when provisioning virtual clusters, which can include configurations for applications and software bundled with Amazon EMR on ::Aws::EKS. A configuration consists of a classification, properties, and optional nested configurations. A classification refers to an application-specific configuration file. Properties are the settings you want to change in that file.
