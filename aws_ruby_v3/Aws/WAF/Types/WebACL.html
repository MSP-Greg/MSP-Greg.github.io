      Overview

      This is AWS ::Aws::WAF Classic documentation. For more information, see AWS ::Aws::WAF Classic in the developer guide.

      For the latest version of AWS WAF, use the AWS WAFV2 API and see the AWS ::Aws::WAF Developer Guide. With the latest version, AWS WAF has a single set of endpoints for regional and global use.

      Contains the Rules that identify the requests that you want to allow, block, or count. In a WebACL, you also specify a default action (ALLOW or BLOCK), and the action for each Rule that you add to a WebACL, for example, block requests from specified IP addresses or block requests from specified referrers. You also associate the WebACL with a ::Aws::CloudFront distribution to identify the requests that you want AWS WAF to filter. If you add more than one Rule to a WebACL, a request needs to match only one of the specifications to be allowed, blocked, or counted. For more information, see UpdateWebACL.

    #metric_name  ⇒ String  (rw)
  
      

      
  
      
    
      A friendly name or description for the metrics for this WebACL.
The name can contain only alphanumeric characters (A-Z, a-z, 0-9),
with maximum length 128 and minimum length one. It can't contain
whitespace or metric names reserved for AWS WAF, including "All"
and "Default_Action." You can't change MetricName after you
create the WebACL.
