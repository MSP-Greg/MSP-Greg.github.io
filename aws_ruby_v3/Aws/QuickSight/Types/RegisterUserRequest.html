    #custom_permissions_name  ⇒ String  (rw)
  
      

      
  
      
    
      (Enterprise edition only) The name of the custom permissions profile
that you want to assign to this user. Customized permissions allows
you to control a user's access by restricting access the following
operations:
      

        

      • 

        Create and update data sources
        

        • 

      • 

        Create and update datasets
        

        • 

      • 

        Create and update email reports
        

        • 

      • 

        Subscribe to email reports
        

        • 

      

      To add custom permissions to an existing user, use UpdateUser
instead.
      

      A set of custom permissions includes any combination of these
restrictions. Currently, you need to create the profile names for
custom permission sets by using the ::Aws::QuickSight console. Then, you
use the RegisterUser API operation to assign the named set of
permissions to a ::Aws::QuickSight user.
      

      ::Aws::QuickSight custom permissions are applied through ::Aws::IAM policies.
Therefore, they override the permissions typically granted by
assigning ::Aws::QuickSight users to one of the default security cohorts in
::Aws::QuickSight (admin, author, reader).
      

      This feature is available only to ::Aws::QuickSight Enterprise edition
subscriptions that use SAML 2.0-Based Federation for Single Sign-On
(SSO).

    #identity_type  ⇒ String  (rw)
  
      

      
  
      
    
      Amazon QuickSight supports several ways of managing the identity of
users. This parameter accepts two values:
      

        

      • 

        IAM: A user whose identity maps to an existing ::Aws::IAM user or
role.
        

        • 

      • 

        QUICKSIGHT: A user whose identity is owned and managed
internally by Amazon QuickSight.

    #session_name  ⇒ String  (rw)
  
      

      
  
      
    
      You need to use this parameter only when you register one or more
users using an assumed ::Aws::IAM role. You don't need to provide the
session name for other scenarios, for example when you are
registering an ::Aws::IAM user or an Amazon QuickSight user. You can
register multiple users using the same ::Aws::IAM role if each user has a
different session name. For more information on assuming ::Aws::IAM roles,
see  assume-role  in the AWS CLI Reference.

      The Amazon ::Aws::QuickSight role for the user. The user role can be one of

      • 

        READER: A user who has read-only access to dashboards.
