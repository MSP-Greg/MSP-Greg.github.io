      Overview

      Note:

      When making an API call, you may pass CompleteMultipartUploadInput data as a hash:

      {
  account_id: "string", # required
  vault_name: "string", # required
  upload_id: "string", # required
  archive_size: 1,
  checksum: "string",
}

      Provides options to complete a multipart upload operation. This informs Amazon Glacier that all the archive parts have been uploaded and Amazon S3 ::Aws::Glacier (Glacier) can now assemble the archive from the uploaded parts. After assembling and saving the archive to the vault, ::Aws::Glacier returns the URI path of the newly created archive resource.

      Instance Attribute Details

      #account_id ⇒ String (rw)

      The AccountId value is the AWS account ID of the account that owns the vault. You can either specify an AWS account ID or optionally a single '-' (hyphen), in which case Amazon S3 ::Aws::Glacier uses the AWS account ID associated with the credentials used to sign the request. If you use an account ID, do not include any hyphens ('-') in the ID.

    #checksum  ⇒ String  (rw)
  
      

      
  
      
    
      The SHA256 tree hash of the entire archive. It is the tree hash of
SHA256 tree hash of the individual parts. If the value you specify
in the request does not match the SHA256 tree hash of the final
assembled archive as computed by Amazon S3 ::Aws::Glacier (Glacier),
::Aws::Glacier returns an error and the request fails.
