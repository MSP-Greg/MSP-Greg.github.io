    #deliver_logs_error_message  ⇒ String  (rw)
  
      

      
  
      
    
      Information about the error that occurred. Rate limited indicates
that ::Aws::CloudWatch Logs throttling has been applied for one or more
network interfaces, or that you've reached the limit on the number
of log groups that you can create. Access error indicates that the
::Aws::IAM role associated with the flow log does not have sufficient
permissions to publish to ::Aws::CloudWatch Logs. Unknown error indicates
an internal error.

    #log_destination  ⇒ String  (rw)
  
      

      
  
      
    
      Specifies the destination to which the flow log data is published.
Flow log data can be published to an ::Aws::CloudWatch Logs log group or an
Amazon S3 bucket. If the flow log publishes to ::Aws::CloudWatch Logs, this
element indicates the Amazon Resource Name (ARN) of the ::Aws::CloudWatch
Logs log group to which the data is published. If the flow log
publishes to Amazon S3, this element indicates the ::Aws::ARN of the Amazon
::Aws::S3 bucket to which the data is published.

    #max_aggregation_interval  ⇒ Integer  (rw)
  
      

      
  
      
    
      The maximum interval of time, in seconds, during which a flow of
packets is captured and aggregated into a flow log record.
      

      When a network interface is attached to a Nitro-based instance,
the aggregation interval is always 60 seconds (1 minute) or less,
regardless of the specified value.
      

      Valid Values: 60 | 600
