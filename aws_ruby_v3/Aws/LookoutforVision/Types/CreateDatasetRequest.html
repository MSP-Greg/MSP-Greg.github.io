      Instance Attribute Details

      #client_token ⇒ String (rw)

      ClientToken is an idempotency token that ensures a call to CreateDataset completes only once. You choose the value to pass. For example, An issue, such as an network outage, might prevent you from getting a response from CreateDataset. In this case, safely retry your call to CreateDataset by using the same ClientToken parameter value. An error occurs if the other input parameters are not the same as in the first request. Using a different value for ClientToken is considered a new call to CreateDataset. An idempotency token is active for 8 hours.

      A suitable default value is auto-generated. You should normally not need to pass this option.

    #dataset_source  ⇒ Types::DatasetSource  (rw)
  
      

      
  
      
    
      The location of the manifest file that Amazon Lookout for Vision
uses to create the dataset.
      

      If you don't specify DatasetSource, an empty dataset is created
and the operation synchronously returns. Later, you can add JSON
Lines by calling UpdateDatasetEntries.
      

      If you specify a value for DataSource, the manifest at the ::Aws::S3
location is validated and used to create the dataset. The call to
CreateDataset is asynchronous and might take a while to complete.
To find out the current status, Check the value of Status returned
in a call to DescribeDataset.
