      Overview

      Note:

      When making an API call, you may pass BuildConfiguration data as a hash:

      {
  artifact_name: "String",
  code_build_service_role: "NonEmptyString", # required
  compute_type: "BUILD_GENERAL1_SMALL", # accepts BUILD_GENERAL1_SMALL, BUILD_GENERAL1_MEDIUM, BUILD_GENERAL1_LARGE
  image: "NonEmptyString", # required
  timeout_in_minutes: 1,
}

      Instance Attribute Summary

      • #artifact_name ⇒ String rw

        The name of the artifact of the ::Aws::CodeBuild build.

      • #code_build_service_role ⇒ String rw

        The Amazon ::Aws::ElasticBeanstalk::Resource Name (ARN) of the AWS Identity and Access Management (IAM) role that enables AWS CodeBuild to interact with dependent AWS services on behalf of the AWS account.

      • #compute_type ⇒ String rw

        Information about the compute resources the build project will use.

      • #image ⇒ String rw

        The ID of the Docker image to use for this build project.

      • #timeout_in_minutes ⇒ Integer rw

        How long in minutes, from 5 to 480 (8 hours), for AWS CodeBuild to wait until timing out any related build that does not get marked as completed.

      Instance Attribute Details

      #artifact_name ⇒ String (rw)

      The name of the artifact of the ::Aws::CodeBuild build. If provided, Elastic Beanstalk stores the build artifact in the ::Aws::S3 location S3-bucket/resources/application-name/codebuild/codebuild-version-label-artifact-name.zip. If not provided, Elastic Beanstalk stores the build artifact in the ::Aws::S3 location S3-bucket/resources/application-name/codebuild/codebuild-version-label.zip.
