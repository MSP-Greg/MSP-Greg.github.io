      The payload ::Aws::Kinesis Video Streams returns is a sequence of chunks
from the specified stream. For information about the chunks, see .
The chunks that ::Aws::Kinesis Video Streams returns in the GetMedia call
also include the following additional Matroska (MKV) tags:
      

        

      • 

        AWS_KINESISVIDEO_CONTINUATION_TOKEN (UTF-8 string) - In the
event your GetMedia call terminates, you can use this
continuation token in your next request to get the next chunk
where the last request terminated.
        

        • 

      • 

        AWS_KINESISVIDEO_MILLIS_BEHIND_NOW (UTF-8 string) - ::Aws::KinesisVideoMedia::Client
applications can use this tag value to determine how far behind
the chunk returned in the response is from the latest chunk on the
stream.
        

        • 

      • 

        AWS_KINESISVIDEO_FRAGMENT_NUMBER - Fragment number returned in
the chunk.
        

        • 

      • 

        AWS_KINESISVIDEO_SERVER_TIMESTAMP - Server timestamp of the
fragment.
        

        • 

      • 

        AWS_KINESISVIDEO_PRODUCER_TIMESTAMP - Producer timestamp of the
fragment.
        

        • 

      

      The following tags will be present if an error occurs:
      

        

      • 

        AWS_KINESISVIDEO_ERROR_CODE - String description of an error
that caused GetMedia to stop.
        

        • 

      • 

        AWS_KINESISVIDEO_ERROR_ID: Integer code of the error.
        

        • 

      

      The error codes are as follows:
      

        

      • 

        3002 - Error writing to the stream
        

        • 

      • 

        4000 - Requested fragment is not found
        

        • 

      • 

        4500 - Access denied for the stream's ::Aws::KMS key
        

        • 

      • 

        4501 - Stream's ::Aws::KMS key is disabled
        

        • 

      • 

        4502 - Validation error on the stream's ::Aws::KMS key
        

        • 

      • 

        4503 - ::Aws::KMS key specified in the stream is unavailable
        

        • 

      • 

        4504 - Invalid usage of the ::Aws::KMS key specified in the stream
        

        • 

      • 

        4505 - Invalid state of the ::Aws::KMS key specified in the stream
        

        • 

      • 

        4506 - Unable to find the ::Aws::KMS key specified in the stream
        

        • 

      • 

        5000 - Internal error
