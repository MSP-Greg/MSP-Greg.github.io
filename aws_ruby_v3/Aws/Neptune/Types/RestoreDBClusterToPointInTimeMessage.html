    #kms_key_id  ⇒ String  (rw)
  
      

      
  
      
    
      The AWS ::Aws::KMS key identifier to use when restoring an encrypted DB
cluster from an encrypted DB cluster.
      

      The KMS key identifier is the Amazon Resource Name (ARN) for the ::Aws::KMS
encryption key. If you are restoring a DB cluster with the same AWS
account that owns the ::Aws::KMS encryption key used to encrypt the new DB
cluster, then you can use the ::Aws::KMS key alias instead of the ::Aws::ARN for
the ::Aws::KMS encryption key.
      

      You can restore to a new DB cluster and encrypt the new DB cluster
with a ::Aws::KMS key that is different than the ::Aws::KMS key used to encrypt
the source DB cluster. The new DB cluster is encrypted with the ::Aws::KMS
key identified by the KmsKeyId parameter.
      

      If you do not specify a value for the KmsKeyId parameter, then the
following will occur:
      

        

      • 

        If the DB cluster is encrypted, then the restored DB cluster is
encrypted using the KMS key that was used to encrypt the source DB
cluster.
        

        • 

      • 

        If the DB cluster is not encrypted, then the restored DB cluster
is not encrypted.
        

        • 

      

      If DBClusterIdentifier refers to a DB cluster that is not
encrypted, then the restore request is rejected.

      The type of restore to be performed. You can specify one of the

        full-copy - The new DB cluster is restored as a full copy of the
