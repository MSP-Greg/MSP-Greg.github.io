      Overview

      Note:

      When making an API call, you may pass APNSVoipChannelRequest data as a hash:

      {
  bundle_id: "__string",
  certificate: "__string",
  default_authentication_method: "__string",
  enabled: false,
  private_key: "__string",
  team_id: "__string",
  token_key: "__string",
  token_key_id: "__string",
}

      Specifies the status and settings of the APNs (Apple Push Notification service) VoIP channel for an application.

      Instance Attribute Summary

      • #bundle_id ⇒ String rw

        The bundle identifier that's assigned to your iOS app.

      • #certificate ⇒ String rw

        The APNs client certificate that you received from Apple, if you want Amazon Pinpoint to communicate with APNs by using an APNs certificate.

      • #default_authentication_method ⇒ String rw

        The default authentication method that you want Amazon Pinpoint to use when authenticating with APNs, key or certificate.

      • #enabled ⇒ Boolean rw

        Specifies whether to enable the APNs VoIP channel for the application.

      • #private_key ⇒ String rw

        The private key for the APNs client certificate that you want Amazon Pinpoint to use to communicate with APNs.

      • #team_id ⇒ String rw

        The identifier that's assigned to your Apple developer account team.

      • #token_key ⇒ String rw

        The authentication key to use for APNs tokens.

      • #token_key_id ⇒ String rw

        The key identifier that's assigned to your APNs signing key, if you want Amazon Pinpoint to communicate with APNs by using APNs tokens.
