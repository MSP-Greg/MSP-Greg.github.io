      Overview

      When CodePipeline returns an error response, the Ruby SDK constructs and raises an error. These errors all extend Aws::CodePipeline::Errors::ServiceError < ::Aws::Errors::ServiceError

      You can rescue all ::Aws::CodePipeline errors using ServiceError:

      begin
  # do stuff
rescue Aws::CodePipeline::Errors::ServiceError
  # rescues all CodePipeline API errors
end

      Request Context

      ServiceError objects have a #context method that returns information about the request that generated the error. See ::Seahorse::Client::RequestContext for more information.
