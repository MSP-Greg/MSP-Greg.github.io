      Overview

      A distribution tells ::Aws::CloudFront where you want content to be delivered from, and the details about how to track and manage content delivery.

      #active_trusted_key_groups ⇒ Types::ActiveTrustedKeyGroups (rw)

      ::Aws::CloudFront automatically adds this field to the response if you’ve configured a cache behavior in this distribution to serve private content using key groups. This field contains a list of key groups and the public keys in each key group that ::Aws::CloudFront can use to verify the signatures of signed URLs or signed cookies.

    #active_trusted_signers  ⇒ Types::ActiveTrustedSigners  (rw)
  
      

      
  
      
    
      We recommend using TrustedKeyGroups instead of TrustedSigners.
      

      ::Aws::CloudFront automatically adds this field to the response if you’ve
configured a cache behavior in this distribution to serve private
content using trusted signers. This field contains a list of AWS
account IDs and the active ::Aws::CloudFront key pairs in each account that
::Aws::CloudFront can use to verify the signatures of signed URLs or signed
cookies.

    #alias_icp_recordals  ⇒ Array<Types::AliasICPRecordal>  (rw)
  
      

      
  
      
    
      AWS services in China customers must file for an Internet Content
Provider (ICP) recordal if they want to serve content publicly on an
alternate domain name, also known as a CNAME, that they've added to
::Aws::CloudFront. AliasICPRecordal provides the ICP recordal status for
CNAMEs associated with distributions.
      

      For more information about ICP recordals, see  Signup, Accounts,
and Credentials in Getting Started with AWS services in China.
