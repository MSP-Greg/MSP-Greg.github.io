      Note:

      When making an API call, you may pass WorkspaceCreationProperties data as a hash:

  enable_work_docs: false,
  enable_internet_access: false,
  default_ou: "DefaultOu",
  custom_security_group_id: "SecurityGroupId",
  user_enabled_as_local_administrator: false,

      Describes the default properties that are used for creating ::Aws::WorkSpaces. For more information, see Update Directory Details for Your WorkSpaces.

    #default_ou  ⇒ String  (rw)
  
      

      
  
      
    
      The default organizational unit (OU) for your ::Aws::WorkSpaces
directories. This string must be the full Lightweight Directory
Access Protocol (LDAP) distinguished name for the target domain and
OU. It must be in the form "OU=value,DC=value,DC=value", where
value is any string of characters, and the number of domain
components (DCs) is two or more. For example,
OU=WorkSpaces_machines,DC=machines,DC=example,DC=com.
      

        

      • 

        To avoid errors, certain characters in the distinguished name must
be escaped. For more information, see  Distinguished Names in
the Microsoft documentation.
        

        • 

      • 

        The API doesn't validate whether the OU exists.

    #enable_work_docs  ⇒ Boolean  (rw)
  
      

      
  
      
    
      Indicates whether Amazon WorkDocs is enabled for your ::Aws::WorkSpaces.
      

       If WorkDocs is already enabled for a ::Aws::WorkSpaces directory and you
disable it, new ::Aws::WorkSpaces launched in the directory will not have
::Aws::WorkDocs enabled. However, ::Aws::WorkDocs remains enabled for any existing
::Aws::WorkSpaces, unless you either disable users' access to ::Aws::WorkDocs or
you delete the ::Aws::WorkDocs site. To disable users' access to ::Aws::WorkDocs,
see Disabling Users in the Amazon ::Aws::WorkDocs Administration
Guide. To delete a ::Aws::WorkDocs site, see Deleting a Site in the
Amazon ::Aws::WorkDocs Administration Guide.
      

      If you enable WorkDocs on a directory that already has existing
::Aws::WorkSpaces, the existing ::Aws::WorkSpaces and any new ::Aws::WorkSpaces that are
launched in the directory will have ::Aws::WorkDocs enabled.
