      Instance Attribute Details

      #data_retention_in_hours ⇒ Integer (rw)

      The number of hours that you want to retain the data in the stream. ::Aws::Kinesis Video Streams retains the data in a data store that is associated with the stream.

      The default value is 0, indicating that the stream does not persist data.

      When the DataRetentionInHours value is 0, consumers can still consume the fragments that remain in the service host buffer, which has a retention time limit of 5 minutes and a retention memory limit of 200 MB. Fragments are removed from the buffer when either limit is reached.

::Aws::Kinesis Video Streams to use to encrypt stream data.
