      Overview

      mutex_m.rb

      When ‘mutex_m’ is required, any object that extends or includes Mutex_m will be treated like a Mutex.

      Start by requiring the standard library Mutex_m:

      require "mutex_m"

      From here you can extend an object with Mutex instance methods:

      obj = Object.new
obj.extend Mutex_m

      Or mixin Mutex_m into your module to your class inherit Mutex instance methods — remember to call super() in your class initialize method.

      class Foo
  include Mutex_m
  def initialize
    # ...
    super()
  end
  # ...
end
obj = Foo.new
# this obj can be handled like Mutex

def Mutex_m.define_aliases(cl) # :nodoc:
  cl.alias_method(:locked?, :mu_locked?)
  cl.alias_method(:lock, :mu_lock)
  cl.alias_method(:unlock, :mu_unlock)
  cl.alias_method(:try_lock, :mu_try_lock)
  cl.alias_method(:synchronize, :mu_synchronize)
end

def mu_extended # :nodoc:
  unless (defined? locked? and
          defined? lock and
          defined? unlock and
          defined? try_lock and
          defined? synchronize)
    Mutex_m.define_aliases(singleton_class)
  end
  mu_initialize
end
